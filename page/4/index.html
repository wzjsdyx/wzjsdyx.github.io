0、时钟只要有负载，就需要插入scan chain，换言之，就需要scan clock；

1、针对clock_mux，如果其选择信号来自于寄存器，需要使用test_scan_mode进行控制（因为在launch测试向量的时候，会导致clock乱动）

DFT测试相关的PAD

内部设计中涉及到的相关信号

1、test_se 连接外部PAD；即scan enable;用于控制shift和capture;
一般由工具自动trace并连接到test point上；

  • dft use to connect DFF SE PIN;
  • RTL not use;
    一般由工具自动trace并连接到test point上

2、test mode 来自trapping；

  • 控制测试mode下的电压等；
  • 控制cm0p_reset_release
  • top_ckgen中切换到scan_clk;
  • 控制pinmux

3、dft会使用这个test mode信号吗？
会使用到这个信号，需要切换pinmux，控制PAD的功能为DFT相关；

4、为什么有些用test mode切换？有些用test scan mode切换？

  • 只有ATPG测试的时候，test scan mode才会拉高；
  • mbist测试的时候（test mode会拉高，test scan mode不会拉高）也需要使用DFT的时钟，使用test mode进行切换；
  • module内部寄存器如果只是为了ATPG测试的话，一般用test scan mode进行时钟切换，而不是test mode；（一般只有ckgen才会用test mode）
    顺序为：test mode-》IJTAG-》TDR寄存器-》test_scan_mode在整个ATPG阶段一直为高电平；

5、test scan mode信号一般由DFT logic 生成

时钟源与时钟架构

时钟架构：

  • 内部时钟：芯片内部的模拟电路OSC生成的时钟源；

  • 外部时钟：芯片外部灌入时钟，可能经过芯片内部的模拟电路XOSC，也可能不经过芯片内部的模拟电路；

HSI:high speed internal clock
HSE:high speed external clock

LSI:low speed internal clock
LSE:low speed external clock

VHSI:very high speed internal clock

补充知识：

🔁 一、PLL 在芯片中的工作原理

1. 作用：

  • 提供高频、低抖动、精确的系统时钟
  • 常用于 CPU、DSP、USB、DDR、SerDes 等需要高速时钟的模块

2. 工作步骤：

  1. 接收参考时钟（如外部晶振的 25MHz）
  2. 锁相环倍频：通过电压控制振荡器 VCO(Voltage-Controlled Oscillator)和反馈分频器FBKDIV((Feedback Divider)将时钟升频为 1GHz 或更高
  3. 稳定锁相：通过相位比较器不断校正，使输出与参考时钟频率/相位保持一致
  4. 输出多个分频版本：不同子系统可能需要不同的频率（如 500MHz、250MHz）

🔋 二、LDO 在芯片中的工作原理

1. 作用：

  • 将较高电压（如 3.3V）稳定地转换为较低电压（如 1.1V）
  • 提供给数字逻辑、模拟电路、PLL、Memory 等不同模块
  • 特别适合低噪声/模拟场景，如音频电路、RF 前端、ADC/DAC

2. 工作机制：

  • 检测输出电压是否达到目标值
  • 如果过高，控制管减小输出；过低则增加输出
  • 保持输出电压稳定，不受输入电压/负载变化影响

🔗 三、PLL 与 LDO 的联系（非常关键）

✅ 1. PLL 通常需要独立 LDO 供电

  • PLL 对电源噪声特别敏感，稍有干扰会导致抖动增加、锁相失败
  • 所以芯片中常有专门的“PLL LDO”或“Analog LDO”为 PLL 提供干净稳定的电源
  • 某些高端芯片甚至会使用多个 LDO，为不同的 PLL（CPU PLL、DDR PLL）分别供电

✅ 2. LDO 为 PLL 输出频率的稳定性提供保障

  • 如果电源波动，PLL VCO 的频率也会漂移
  • 使用 LDO 可以减少这些波动，提升频率精度和锁定性能

BUS protection机制

发表于 2025-05-27 分类于 ARM ， Cortex-R52 Valine：

1. UTID 的核心功能

  • 定义：UTID 是主设备（Master）在 AXI 地址通道（Address Channel）上生成的唯一事务标识符，用于唯一标识系统中每个进行中的 AXI 事务（Outstanding Transaction）。
  • 核心作用：
    • 事务追踪：确保互连（Interconnect）能够正确路由事务并跟踪其生命周期（发起、传输、响应）。
    • 错误检测：通过验证 UTID 的全局唯一性和一致性，检测互连控制逻辑中的错误（如事务丢失、重复或路由混乱）。

2. UTID 的生成规则

(1) 主设备责任

  • 唯一性要求：
    • 同一主设备：同一主设备（Master Interface）生成的每个进行中事务（Outstanding Transaction）的 UTID 必须不同。
    • 不同主设备：Cortex-R52 处理器的多个主设备接口之间，输出的 UTID 必须互不相同。
    • 系统级唯一性：系统中其他主设备（非 Cortex-R52 的主设备）生成的 UTID 不得与 Cortex-R52 生成的 UTID 冲突。
  • 设计约束：
    • 系统设计时需为每个主设备分配独立的 UTID 生成范围（例如分区或掩码机制）。
    • UTID 的生成算法需保证在最大事务并发数内不重复（例如递增计数器结合主设备 ID）。

(2) 从设备责任

  • 响应一致性：
    • 从设备（Slave）在响应事务时，必须返回与主设备原始 UTID 完全相同的值。
    • 若从设备修改或丢失 UTID，互连保护机制会触发错误（如事务终止或异常上报）。

3. UTID 与 AXI ID 的关系

  • AXI ID 的作用：
    • AXI 协议中，ID 信号用于区分同一主设备发起的多个乱序事务。
    • AXI ID 可被主设备复用（例如同一 ID 在不同时间段用于不同事务）。
  • UTID 的增强特性：
    • 全局唯一性：无论 AXI ID 是否复用，每个事务的 UTID 必须在全系统范围内唯一。
    • 独立于 AXI ID：UTID 与 AXI ID 无直接关联，二者服务于不同目的：
      • AXI ID：管理事务乱序完成。
      • UTID：确保事务路由和逻辑正确性。

4. R52 UTID的构建过程

Transaction ID 不是 AXI ID，两者在功能和范围上有本质区别：

1. 定义与用途

  • Transaction ID（UTID[3:0]）
    • 功能：作为 UTID（唯一事务标识符） 的一部分，用于在 特定端口内 唯一标识每个未完成事务（Outstanding Transaction）。
    • 范围：仅在所属端口的未完成事务中唯一，结合 Cluster ID、Core ID、Port ID 共同实现全局唯一性。
    • 目的：支持互连保护机制，检测事务丢失、重复或路由错误。
  • AXI ID（如 AWID/ARID）
    • 功能：AXI 协议中用于标识事务的字段，支持乱序完成（Out-of-Order Completion）。
    • 范围：通常仅在 同一主设备内 唯一，不同主设备可使用相同 ID。
    • 目的：允许从设备返回响应时正确匹配事务。

2. 结构与生成规则

  • Transaction ID
    • 结构：UTID[3:0]，4 位宽度，取值范围 0-15。
    • 生成规则：由端口内部维护的计数器或分配器生成，确保端口内未完成事务的 ID 不重复。
  • AXI ID
    • 结构：AXI 协议定义的字段（如 4 位或更多位宽），由主设备自行分配。
    • 生成规则：主设备可复用 ID（例如同一 ID 在不同时间段用于不同事务），仅需保证同一主设备内未完成事务的 ID 唯一。

outstanding和out-of-order理解

发表于 2025-05-08 分类于 ARM ， AMBA ， AXI Valine：

outstanding的概念：

支持多个超发事务（Multiple Outstanding）

  • 含义：总线允许在未收到前一个事务的响应之前，连续发起多个新的事务请求。
  • 优势：提升总线利用率，避免因等待响应而空闲。例如，在访问高延迟设备（如内存）时，总线可以持续发送读写请求，无需阻塞。
  • 实现：通过独立的地址/数据通道（如AXI的读/写地址、读/写数据通道）分离请求和响应，实现流水线化操作。
