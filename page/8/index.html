常见颜色设置

ANSI 颜色代码分为两部分：前景色（文字颜色） 和 背景色。

颜色 代码（前景色） 代码（背景色） 示例
黑色 30 40 \033[1;30m 黑色文字
红色 31 41 \033[1;31m 红色文字
绿色 32 42 \033[1;32m 绿色文字
黄色 33 43 \033[1;33m 黄色文字
蓝色 34 44 \033[1;34m 蓝色文字
品红色 35 45 \033[1;35m 品红文字
青色 36 46 \033[1;36m 青色文字
白色 37 47 \033[1;37m 白色文字

设置方法示例

代码说明

  1. \033 或 \x1B：
    • ANSI 转义序列的起始符，告诉终端接下来是控制命令。
  2. [1;36m：
    • 1：设置文本为高亮（粗体）。
    • 36：颜色代码，表示青色（cyan）。
  3. *******************************************************：
    • 打印的实际文本。
  4. \033[0m：
    • 终止颜色控制，将后续文本恢复到默认颜色

访问类型定义

Word 32 位 uint32_t 或 int 访问四个字节

实现方式

通过指针类型来实现不同的访问大小。例如，假设一个外设寄存器地址为 0x40000000。

verilog timescale

发表于 2024-12-04 更新于 2024-12-05 分类于 verilog语法 Valine：

xrun工具编译的文件一定要有timescale的设置

  • 如果 有-timescale编译选项：如果verilog文件没有timescale定义，会使用-timescale指定的默认timescale配置；

  • 如果没有-timescale编译选项：后面的verilog文件如果没有timescale定义，会继承前面verilog文件的timescale配置；

xrun工具相关的命令选项：

-timescale：

xrun -timescale ‘<time_unit/time_precision>’

如果verilog文件没有timescale定义，会使用-timescale指定的默认timescale配置

-override_timescale：

xrun -timescale <time_unit/time_precision> -override_timescale

-vtimescale:

xrun -vtimescale time_unit/time_precision

xrun工具timescale配置优先级

文件定义>-timescale编译命令>-vtimescale编译命令

编译器timescale和module timescale

编译器是以design中最小的timescale作为仿真精度；

但是模块内部还是以模块自身的timescale进行计算；

-default_ext verilog

Functional Description：
xrun generates an error if it encounters a file with an extension that it does not recognize. Use the -default_ext option to specify the file type for files with unrecognized file extensions.

For example, the following specifies that all files with unrecognized extensions are to be treated as Verilog files:

-default_ext verilog

-vlog_ext

Syntax

% xrun *_ext [+].[,.…]

Example 1

% xrun -vlog_ext .v,.rtl,.vg

This example specifies that .v, .rtl, and .vg are valid extensions for Verilog files. The default file extensions for Verilog files are .v, .vp, .vs, .V, .VP, and .VS. With the command used in this example, the list of built-in, predefined extensions is replaced with a new list that is provided in the command.

Example 2

% xrun -vlog_ext +.rtl,.vg

This example adds extensions to the list of built-in, predefined extensions by using a plus sign (+) argument. Two new options, .rtl and .vg, are added.
This is same as using:

% xrun -vlog_ext .v,.vp,.vs,.V,.VP,.VS,.rtl,.vg

Example 3

% xrun -sysv_ext +, mysvtop

This example specifies the file type SystemVerilog for file mysvtop, which does not have any extension.

-loadpli1选项说明：

背景知识

PLI（Programming Language Interface） 是 Verilog 仿真中的一种扩展机制，允许用户通过 C 语言编写自定义的系统任务或函数。这些任务或函数可以在仿真运行时被调用，用于执行仿真器本身无法直接实现的特殊功能。

关键点解析

  1. 动态共享库加载：
  • 如果一个 PLI 应用程序已经被编译为动态共享库（比如 .so 或 .dll 文件），可以通过 -loadpli1 选项在运行时加载这个共享库。
  • 加载的同时，这个选项还会注册 PLI 应用程序中定义的新系统任务。
  1. 引导函数（Bootstrap Function）：
  • 每个 PLI 应用程序共享库中包含一个引导函数，负责将该应用程序中的新系统任务注册到仿真器。
  • 通过 -loadpli1 指定共享库的同时，必须提供该引导函数的名称。
  • 格式：库名称:引导函数名。
  1. 支持多个共享库：
  • 可以一次性加载多个 PLI 应用程序。
  • 用逗号分隔多个引导函数名，注意：逗号之间不能有空格。
  1. 文件扩展名处理：
  • 动态共享库的文件扩展名（如 .so 或 .dll）可以省略。
  • 仿真器会根据操作系统自动补充扩展名。例如：
    • 在 Linux 或 Solaris 上扩展为 .so。
    • 在 Windows 上扩展为 .dll。
  1. PLI1.0 应用：
  • 对于 PLI 1.0 应用，仿真器总是按照上述流程加载共享库并执行提供的引导函数。

By default, models are compiled with performance optimizations turned on. Some of these optimizations disable the ability to set line breakpoints and to single-step though source code. Use -linedebug to override this behavior. Running the simulator with the SimVision debug environment (xmsim -gui) then lets you look at the source code and see where the simulator has stopped for a line breakpoint or when you are stepping through the code.

Using this option sets the default access to simulation objects to read/write/connectivity when the design is elaborated. Do not use this option if you want to run in regression mode.

-nontcglitch

Functional Description

In a design that contains negative timing checks, the default negative timing check algorithm will calculate, when needed, delays with two values (rise and fall) for the different nets. When a delay with two values is calculated, there is the possibility that an event on the input net may cancel a scheduled event on the internal signal driven by the delay. This is called glitch suppression. Because glitch suppression can hide input events from a timing check’s input, the simulator generates a glitch suppression timing violation if an event on a delayed signal is canceled.

Glitch suppression messages are displayed by default. Use the xmsim -nontcglitch option if you want to suppress these messages.

VCS选项的替换

编译选项

主要选项

+v2k

  • 启用 Verilog-2001 语言特性。

-sverilog

  • 启用 SystemVerilog 支持。

+verilog2001ext+.vp

  • 指定 Verilog-2001 的文件扩展名为 .vp。

-timescale=1ns/10ps

  • 指定仿真时间精度为 1ns 和最小时间单位为 10ps。

-q

  • 安静模式，减少非必要的日志输出。

-notice

  • 启用 VCS 通知信息的输出。

+lint=...

  • 开启 lint 检查以报告潜在的问题：
    • all：启用所有 lint 检查。
    • noSVA-NSVU 等：禁用特定类型的检查。
    • 例如，noVCDE 表示禁用对未使用的变量的警告。

+warn=noPHNE

  • 禁用特定的警告，如未连接的端口。

-debug_access+all

  • 启用所有调试访问功能（例如变量值、波形信号）。

-debug_region=cell+lib

  • 指定调试级别，包括 cell 和 lib 范围内的信号。

-j16

  • 启用多线程编译，设置线程数为 16。

-full64

  • 启用 64 位模式。

-kdb

  • 启用 Kernel Database（KDB）功能，用于动态调试。

VPI 相关选项

+vpi

  • 使能vpi

-LDFLAGS "-z muldefs"

  • 链接器选项，允许多重定义的符号（-z muldefs 是 Linux 链接选项）。

-P .../jtag_vpi.tab

  • 指定 VPI 的模块定义表（.tab 文件），用于配置 VPI。

-CC "-DVCS_VPI" ...

  • 向 C 预处理器传递宏定义选项 -DVCS_VPI，并编译指定的 C 文件。
