### ✅ 一、芯片功能安全（Functional Safety）

#### 🌟 定义：

<font color=blue>指的是芯片在**发生故障**时，能否**检测并控制故障**，防止产生危险的后果。</font>

#### 📌 关键点：

- 关注“失效是否被检测和控制”
- 基于 ISO 26262 标准（车规芯片）
- 通常涉及：冗余、电压监测、故障检测机制（BIST、ECC、WDT）

#### 🔧 芯片中的**功能安全机制**（Functional Safety）

| 功能模块 | 安全设计                     | 原因                            |
| -------- | ---------------------------- | ------------------------------- |
| CPU      | 双核锁步（Lockstep）         | 保证指令执行正确，有错误可检测  |
| 电源监测 | 电压/电流异常监测            | 防止供电异常造成芯片误动作      |
| 存储器   | ECC（Error Correction Code） | 自动纠正 SRAM/Flash 的 bit 错误 |
| 时钟模块 | 时钟监控器 + 看门狗          | 防止振荡器停止或异常时芯片卡死  |

#### 📎 举例：

一个车载 MCU 控制刹车系统，如果某个电路发生故障（比如时钟停止工作），芯片能快速检测到并进入安全模式（例如紧急制动），避免事故发生。





### 🧩总结对比表：功能安全和信息安全



| 项目     | 功能安全（Functional Safety） | 信息安全（Information Security） |
| -------- | ----------------------------- | -------------------------------- |
| 目标     | 避免因**故障**导致的危险      | 防止**非法访问/攻击**            |
| 面向问题 | 随机硬件失效、系统错误        | 黑客攻击、数据泄露、未授权访问   |
| 典型标准 | ISO 26262                     | ISO/SAE 21434、CC、FIPS 140-3    |
| 常见机制 | ECC、双核锁步、监控模块       | 加密引擎、PUF、Secure Boot       |
| 应用场景 | 刹车、气囊、电池管理系统等    | 车联网通信、OTA升级、电子支付等  |

### 功能安全等级

FMEDA（Failure Modes, Effects and Diagnostic Analysis）

**目的**：定量分析系统在失效下的行为，并计算 **安全指标**，如：

- **SPFM（单点故障指标）**
- **LPFM（潜在故障指标）**
- **PMHF（每小时潜在故障率）**





例如芯片`每个模块有多少晶体管` x `晶体管发生失效的概率` = 可能出现故障点的数量

> 不能的模块有不同的功能安全机制
>
> 例如CPU模块有100个可能的故障点，使用了lockstep安全机制，能覆盖到99%错误（发生错误的时候能上报，进行复位）==》
>
> SRAM 使用ECC，能覆盖90%的错误
>
> 总线
>
> 外设
>
> ...



==>得出芯片整体单点故障指标

如果覆盖率为99+%，那就达到ASIL-D的SPFM标准；

如果覆盖率为90+%，那就达到ASIL-B的SPFM标准；
