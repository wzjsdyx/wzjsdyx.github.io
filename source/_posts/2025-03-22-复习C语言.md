# 编译选项和工具

```bash
CFLAGS="-Wall" make ex1
# 在生成ex1目标的时候，临时启用GCC的-Wall编译选项
```

`CFLAGS`是一个环境变量，用于指定C编译器的编译选项；



工具：

- make

```make
# 定义变量 p_name，如果没有传递则默认为 "main"
p_name ?= main

# clean 目标：删除 p_name 文件
clean:
	rm -f $(p_name)

# com 目标：编译 p_name.c 文件
com: clean
	cc -Wall -g $(p_name).c -o $(p_name)

# run 目标：运行 p_name 可执行文件，并使用 valgrind 检查内存泄漏
run: com
	valgrind --leak-check=full ./$(p_name)

# 默认目标：执行 com
all: com

# 声明伪目标
.PHONY: clean com run all
```

- valgrind
- gdb

# 字符串&字符串数组

对于C语言来说：

- `\0`是一个特殊字符；在存储中的数值为0；字符串常量是以`\0`结尾；%s输出，以`\0`为结束；
- C只是简单地将字符串储存为字符数组，并且在结尾加上`\0`（空字符）

思考如下问题:

```c
char full_name[] = {
    'Z', 'e', 'd',
     ' ', 'A', '.', ' ',
     'S', 'h', 'a', 'w', '\0'
};
printf("The size of full_name (char[]): %ld\n",sizeof(full_name));
printf("%s",full_name);

# 输出:
# 12
# Zed A. Shaw
//====================================================================

char full_name[] = {
    'Z', 'e', 'd',
     ' ', 'A', '.', ' ',
     'S', 'h', 'a', 'w'
};
printf("The size of full_name (char[]): %ld\n",sizeof(full_name));
printf("%s",full_name);

# 输出:
# 12
# Zed A. Shaw
```

> 上面两种结果，sizeof输出的结果符合预期，但是%s的输出怎么解释？
> 🔍 C标准:
> **C标准对字符串的定义如下（C99 §6.4.5）：**
>
> A string literal is a sequence of characters enclosed in double quotation marks, optionally containing escape sequences, and terminated by the null character (\0).
>
> **而在使用 %s 的时候，C标准规定（C17 §7.21.6.1）：**
>
> If no precision is specified, the array shall contain a null character, and the characters are written up to (but not including) the terminating null character.
>
> ==》如果字符数组没有以 `\0` 结尾，`printf` 会继续读取内存，直到偶然遇到一个 `\0`，这可能导致程序崩溃、输出乱码或其他不可预测的行为
>
>  **原因 1：内存布局的巧合**
>
> - 在你的程序中，`full_name` 数组之后的内存恰好是 `\0`。
> - 例如，编译器可能在 `full_name` 数组之后分配了一块初始化为 `0` 的内存，或者 `full_name` 数组之后的内存恰好是 `\0`。
> - 这种情况下，`printf` 会“幸运地”遇到 `\0`，从而正确终止输出。
>
> **原因 3：操作系统的内存管理**
>
> - 操作系统在分配内存时，可能会将新分配的内存初始化为 `0`（尤其是在调试环境中）。
> - 例如，Linux 的 `glibc` 在某些情况下会将堆内存初始化为 `0`。



# C程序结构

## 库函数文件头

```c
#include <stdio.h>
// 在编译之前将标准库的`stdio.h` 头文件内容插入到当前文件中，以便使用标准输入输出函数（如 `printf`）
```

- `#`是预处理指令符号；

- `include`是预处理指令，用于包含头文件；

- `<>`表示编译器在标准库中查找头文件



## main函数的参数

```C
int main(int argc, char *argv[])
```

- `argc`：
  - 表示<font color=blue>命令行参数的数量（argument count）</font>
  - 类型为 `int`。
  - 至少为 `1`，因为第一个参数是程序本身的名称。

- `argv`：

  - 表示<font color=blue>命令行参数的数组（argument vector）</font>

  - 类型为 `char *[]`，即字符串数组。

  - `argv[0]` 是程序名称，`argv[1]` 到 `argv[argc-1]` 是用户传递的参数

```bash
./program arg1 arg2 arg3

int argc = 4;
char *argv[] = {"./program","arg1","arg2","arg3"};
```



## 特殊机制

### NULL



**NULL的定义：**

```C
#if defined (_STDDEF_H) || defined (__need_NULL)
#undef NULL		/* in case <stdio.h> has defined it. */
#ifdef __GNUG__
#define NULL __null
#else   /* G++ */
#ifndef __cplusplus
#define NULL ((void *)0)
#else   /* C++ */
#define NULL 0
#endif  /* C++ */
#endif  /* G++ */
#endif	/* NULL not defined and <stddef.h> or need NULL.  */
#undef	__need_NULL
```

>  ```c
>      #ifdef __GNUG__
>          printf("This is GNU C++ (__GNUG__ is defined)\n");
>      #else
>          printf("This is not GNU C++\n");
>      #endif
>  
>      #ifdef __cplusplus
>          printf("(__cplusplus is defined)\n");
>      #else
>          printf("(__cplusplus is no defined)\n");
>      #endif
>  
>  # 输出
>  (__GNUG__ is no defined)
>  (__cplusplus is no defined)
>  ```
>
> 通过上面的代码判断，编译器中并没有`__GNUG__`也没有`__cplusplus`定义



#### C 的NULL定义：#define NULL ((void *)0)

- **`0`：**
  - 在 C 语言中，`0` 是一个整数类型的数值
- `(void *)`：
  - <font color=blue>`void *` 是一种通用指针类型，可以指向任何类型的数据。</font>
- `(void *)0`：
  - 将整数 `0` 转换为 `void *` 类型的空指针。
  - 空指针表示：<font color=blue>不指向任何有效内存地址的指针</font>
  - 这种表示方式明确<font color=blue>告诉编译器，`NULL` 是一个指针，而不是整数</font>

> 为什么需要 `(void *)0`？
>
> - 类型安全：
>   - 在 C 语言中，<font color=blue>指针和整数是不同的类型。虽然 `0` 可以表示空指针，但直接使用 `0` 可能会导致类型不明确。</font>
>   - 使用 `(void *)0` 可以明确表示 `NULL` 是一个指针。
> - 通用性：
>   - `void *` 是一种通用指针类型，可以隐式转换为任何其他指针类型。
>   - 因此，`(void *)0` 可以安全地用于所有指针类型。



#### C++的NULL定义：#define NULL 0

在 C 语言中，`void *` 是一个“通用指针”，几乎可以隐式转换为其他任意指针类型。

但是C++强调类型安全，`void *` 并不能直接指向其他指针类型

```c
# 举例说明：
int* p = (void*)0;  // C可以，但C++会警告甚至报错（类型不匹配）
```

因此<font color=blue>C++标准中添加了特性：整数 `0`被解释为空指针常量</font>

```c
int* p = 0;  // 在C++中是合法的，等同于“空指针”
```



#### C++11新标准NULL

引入了专用的`空指针常量nullptr`，解决0是整数还是指针的二义性问题；

```c
int i = 0;
int* p = nullptr;
# nullptr 是一种专门的空指针类型（类型是 std::nullptr_t），不再是整数类型。
```
