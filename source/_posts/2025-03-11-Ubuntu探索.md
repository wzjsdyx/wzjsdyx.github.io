# 基本理解



## shell命令和程序

<font color=blue>Shell 命令 是由 shell 直接实现的命令，主要用于控制 shell 环境。</font>

<font color=blue>程序是独立的可执行文件，功能更复杂，用途更广泛。</font>

<font color=blue>使用 `type` 命令可以判断一个命令是 shell 命令还是程序。</font>



### Shell 命令

- **定义**：
  <font color=blue>shell 命令是直接由 shell（如 `bash`、`zsh`）解释和执行的命令。</font>
- **特点**：
  - <font color=blue>通常是 shell 内置的命令</font>
  - 执行速度快，因为<font color=blue>不需要启动新的进程</font>
  - 主要用于控制 shell 的行为或操作 shell 环境。
- **示例**：

| 命令      | 作用                                  |
| :-------- | :------------------------------------ |
| `cd`      | 改变当前工作目录                      |
| `echo`    | 输出文本到终端                        |
| `export`  | 设置环境变量                          |
| `source`  | 执行脚本文件并在当前 shell 环境中生效 |
| `alias`   | 创建命令别名                          |
| `unalias` | 删除命令别名                          |
| `set`     | 设置 shell 选项                       |
| `unset`   | 取消设置变量或函数                    |
| `read`    | 从标准输入读取数据                    |
| `exit`    | 退出当前 shell                        |

### 程序

- **定义**：
  <font color=blue>程序是独立的可执行文件，通常存储在 `/bin`、`/usr/bin` 等目录中。</font>
- **特点**：
  - <font color=blue>由 shell 调用并运行在新的进程中</font>
  - 功能更复杂，通常用于完成特定任务。
  - 可以是系统自带的工具，也可以是用户安装的软件。
- **示例**：

以下是一些常见的程序及其作用：

| 命令    | 作用                   |
| :------ | :--------------------- |
| `ls`    | 列出目录内容           |
| `find`  | 查找文件               |
| `grep`  | 搜索文本               |
| `vim`   | 文本编辑器             |
| `cat`   | 显示文件内容           |
| `mkdir` | 创建目录               |
| `rm`    | 删除文件或目录         |
| `cp`    | 复制文件或目录         |
| `mv`    | 移动或重命名文件或目录 |
| `ps`    | 显示进程状态           |



## 通配符与正则表达式

<font color=blue>一句话总结区别：通配符（wildcards）是文件名匹配模式，正则表达式（regex）是字符串匹配规则</font>

### 通配符（Wildcards）

- **特点**

  - 用在 **Shell（bash）或某些命令（ls、find）** 中。

  - 用来**匹配文件名**。

- **常见通配符**

| 通配符  | 含义                      | 示例                                         |
| ------- | ------------------------- | -------------------------------------------- |
| `*`     | 匹配任意字符（0个或多个） | `*.txt` 匹配 `a.txt`, `b.txt`                |
| `?`     | 匹配任意单个字符          | `?.txt` 匹配 `a.txt`, `b.txt`                |
| `[abc]` | 匹配括号内任意一个字符    | `file[12].txt` 匹配 `file1.txt`, `file2.txt` |

**常见通配符**

```bash
ls *.sh      # 匹配所有以 .sh 结尾的文件
find . -name "*.log"
```

### 正则表达式（Regex）

- **特点**
  - 是 **一种更强大的文本匹配规则**，常用于grep/sed/awk/perl/Python脚本等
- **常见正则符号**

| 正则符  | 含义                       | 示例                           |
| ------- | -------------------------- | ------------------------------ |
| `.`     | 任意一个字符               | `a.c` 匹配 `abc`、`a1c`        |
| `.*`    | 任意长度任意字符           | `a.*c` 匹配 `abc`、`axyzc`     |
| `^`     | 行首                       | `^hello` 匹配以 hello 开头的行 |
| `$`     | 行尾                       | `world$` 匹配以 world 结尾的行 |
| `[a-z]` | 匹配 a 到 z 之间的任意字符 | `[0-9]+` 匹配数字              |

```bash
grep "^error" log.txt       # 匹配以 error 开头的行
grep "file[0-9]\.txt" .     # 匹配 file0.txt ~ file9.txt
```

## shell命令的单引号和双引号

- 单引号

当你使用单引号包裹字符串时，Shell 会**严格地保留其中的字符原样**，这意味着**不对其中的任何特殊字符进行扩展或替换**。特别是其中的 `$`、`*`、`?` 等特殊字符也不会被展开。

- 双引号

当你使用双引号包裹字符串时，Shell 会对其中的一些特殊字符（如 `$`、反引号 等）进行扩展，但是对于 **`\*`**、**`?`** 和 **`[]`** 这些字符，Shell 会 **进行匹配**，而不会被“保留原样”。



## ls/find/grep匹配

ls和find是使用通配符进行匹配，但是ls和find运行时对通配符解释的时机不同；

grep是使用正则表达式进行匹配



> 假设有三个文件：file1.txt  file2.txt  file3.log
>
> 📌  `ls *.txt`
>
> 这个命令运行时，其实是 shell 做了这件事：
>
> ```bash
> ls file1.txt file2.txt
> ```
>
>  `ls` 程序根本看不到 `*.txt`，它接收到的是两个具体文件名，是 **shell 提前帮你匹配好了**。
>
> 
>
>  📌  `find . -name '*.txt'`
>
> 这个命令运行时，shell只是把 `*.txt` 这个字符串原样传给 `find`，find 自己再去查：
>
> ```bash
> 执行的是 find 程序，参数是字符串 '*.txt'
> ```
>
>  **find 内部才去判断**哪些文件名和 `'*.txt'` 匹配，它使用自己的匹配机制（不是正则，是 shell-style 匹配）。
>
> <font color=red>NOTE：`find . -name *.txt`</font>
>
> **不加引号**，shell 会先把 `*.txt` 展开成 `file1.txt file2.txt`
>
> ```bash
> 此时，执行的是find . -name file1.txt file2.txt  # 错了，语法不对
> 所以才说：在 find 中要加引号，这样 shell 就不会抢着处理通配符。
> ```
>
> 
>
> 📌` grep 'D*p'  file3.log`
>
> 这个命令运行时，shell只是把 `a*c` 这个字符串原样传给 `grep`，让grep自己再去做正则匹配；
>
> 通常加引号，防止正则符号被 shell 误解释

## 管道

连接起两个命令的关键就是管道符号`|`. 这一符号的左右都是Shell命令, `A | B`的含义是创建两个进程`A`和`B`, 并将`A`进程的标准输出连接到`B`进程的标准输入；

```bash
$ ll -a | grep "hello"
```



## 重定向

`>`标准输出重定向；

`<`标准输出重定向；

```bash
$ ll -a > filelist.f
# > 是标准输出重定向符号, 可以将前一命令的输出重定向到文件filelist.f, 此时terminal上就无法显示内容；
# 如果既需要重定向到文件中，又想要显示在terminal上，需要使用tee命令
$ ll -a | tee filelist.f
	
# /dev/null是一个特殊的文件，任何输出都此文件中的数据都会被丢弃；
```

## Setting APT source file

修改APT源文件需要root权限；

```shell
# step1：将用户添加到sudo group中
# sudo组是系统中一个特殊的用户组，当一个用户被添加到sudo组后，这个用户就获得了使用sudo命令的权限
# sudo命令会临时提升权限到 root 用户
su -
adduser username sudo  # Change `username` to your user name
exit

# step2：验证用户是否有使用sudo命令的权限
# whoami命令用于显示当前用户的用户名
# sudo whoami，由于sudo会临时提升权限到 root 用户，然后以root用户执行whoami，因此会输出root
whoami
sudo whoami

# step3：修改APT source file
# 也可以直接修改/etc/apt/sources.list文件
sudo sed -i "s/archive.ubuntu.com/mirrors.tuna.tsinghua.edu.cn/g" /etc/apt/sources.list

# step4:确认替换是否成功
cat /etc/apt/sources.list

# step5: 更新软件源
sudo apt-get update

# step6: 安装软件
sudo apt-get install build-essential    # build-essential packages, include binary utilities, gcc, make, and so on
sudo apt-get install man                # on-line reference manual
sudo apt-get install gcc-doc            # on-line reference manual for gcc
sudo apt-get install gdb                # GNU debugger
sudo apt-get install git                # revision control system
sudo apt-get install libreadline-dev    # a library used later
sudo apt-get install libsdl2-dev        # a library used later
```

> - `sed`：
>   - `sed` 是一个流编辑器（stream editor），用于对文本进行处理和转换。
> - `-i`：
>   - 表示直接修改文件内容（in-place edit）。
> - `"s/archive.ubuntu.com/mirrors.tuna.tsinghua.edu.cn/g"`：
>   - `s` 表示替换操作（substitute）。
>   - `archive.ubuntu.com` 是要查找的字符串。
>   - `mirrors.tuna.tsinghua.edu.cn` 是要替换成的字符串。
>   - `g` 表示全局替换（global），即替换文件中所有匹配的字符串。
> - `/etc/apt/sources.list`：
>   - 这是 Ubuntu 系统中存储软件源地址的文件。
>
> ---
>
> `cat` 是 Linux 和 Unix 系统中的一个常用命令，全称是 **concatenate**（连接）：
>
> 它的主要功能是**显示文件内容**，
>
> 也可以用于**创建文件**、**合并文件**等操作
