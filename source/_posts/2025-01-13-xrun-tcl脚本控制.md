> Xcelium仿真器命令语言是基于TCL，能够对模拟器中的对象执行各种操作。

# The Tcl Command-Line Interface

## Tcl Command Syntax

```tcl
xcelium> command [-modifier] [-options] [arguments]
```

## Tcl Command Rules

- 命令的名称必须小写

- 可以使用略缩表达

- 一行可以有多个tcl命令，使用`;`分割

## Command Description Conventions

- **格式符号** 的作用是规范 Tcl 命令的书写和使用，方便用户理解命令中的必填和可选部分。
- 核心规则：
  1. **必须输入的部分**: 使用字面量。
  2. **可选部分**: 用方括号 `[ ]` 标注。
  3. **选择其中一个**: 用竖线 `|` 和大括号 `{ }` 表示。
  4. **重复参数**: 用省略号 `...` 表示，方括号则标明是否可以完全省略。

通过这些符号的配合，Tcl 命令的表达既清晰又灵活，适应多种使用场景。

## TCL Basics

- Comments `#`

- Tcl Commands
- Command Substitution

# Xcelium Simulator Tcl Commands

## database

The Tcl `database` command controls an `SHM, VCD, EVCD `.

> database -open -shm top_sim -default -incsize 500M
>
> probe -create top_sim                                                          -depth 1
> probe -create top_sim.ac7843x                                                  -depth 1
> probe -create top_sim.ac7843x.MTCMOS_wrap                                      -depth 1
> probe -create top_sim.ac7843x.cm4_core                                         -depth 1
> probe -create top_sim.ac7843x.u_flash_whole_top                                -depth 1
> probe -create top_sim.ac7843x.u_flash_whole_top.u_pflash_top.u_pf_bank0_state  -depth all -all -memories
> probe -create top_sim.ac7843x.u_hsm_top_pwr_wrap.hsm_top_pwr_wrap_woiso_u1     -depth all -all -memories
> probe -create top_sim.ac7843x.u_standby_ctl_pwr_wrap.standby_ctl_u1            -depth all -all -memories
> probe -create top_sim.ac7843x.top_design_ctl                                   -depth all -all -memories
> probe -create top_sim.ac7843x.MTCMOS_wrap.MTCMOS.i_i_MTCMOS                    -depth all -all -memories
>
> run
