> 嵌入式软件通常会涉及到程序变量的分配位置，此处做一个总结；

C程序中的数据分为：

- 变量
  - 栈区：由编译器自动分配释放
  - 堆区： 一般由程序员使用malloc函数分配释放，若程序员不释放，程序结束时可能由OS回收
  - 全局/静态变量存储区：存储全局变量以及静态变量（会对全局/静态变量做初始化的和未初始化的区分）

> 在 ARM Cortex-M 处理器上，局部变量的存储位置取决于 **变量的生命周期、大小和编译器优化**。
>
> ARM-M 处理器采用 **通用寄存器和堆栈（Stack）** 来存储局部变量

- 常量（字面常量）
  - 常量存储器：存放字面常量



<font color=blue>程序会将变量搬运到SRAM区域；</font>

<font color=blue>常量仍然会在Flash或者ROM等NVM介质中；</font>

举例说明1：

```c
#include <stdio.h>

// 1. 已初始化的全局变量（会存入 .data 段）
int global_var = 10;

// 2. 已初始化的静态变量（会存入 .data 段）
static int static_var = 20;

// 3. 未初始化的全局变量（存入 .bss 段，不会进入 .data）
int global_uninit_var;

// 4. 未初始化的静态变量（存入 .bss 段，不会进入 .data）
static int static_uninit_var;

// 5. 只读常量数据（存入 .rodata 段）
const char *msg = "Hello, World!";

void test_function() {
    // 6. 局部变量（存入通用寄存器或者栈，不会进入 .data）
    int local_var = 30;

    printf("global_var = %d\n", global_var);
    printf("static_var = %d\n", static_var);
    printf("local_var = %d\n", local_var);
}

int main() {
    test_function();
    return 0;
}

```

举例说明2：

```c
void func() {
    int a = 5;   // a 可能存入寄存器
    int b = 10;  // b 可能存入寄存器
    int c = a + b;
}
编译之后==》
    MOV r0, #5    // 把 a 存入 r0 寄存器
    MOV r1, #10   // 把 b 存入 r1 寄存器
    ADD r2, r0, r1  // c = a + b，r2 保存结果

//=============================================================================================

void func() {
    int x = 100;
    int y = 200;
    int arr[10];  // 数组无法放入寄存器
}

编译之后==》
    SUB sp, sp, #44   // 在栈上分配 44 字节（用于局部变量）
    STR r0, [sp, #0]  // 存储 x
    STR r1, [sp, #4]  // 存储 y
    ADD r2, sp, #8    // arr[0] 的地址

//=============================================================================================
局部变量 优先存入寄存器，但如果变量过多或太大，就会存入栈。
数组、结构体、指针等数据类型一般存入栈，因为它们无法完全放入寄存器
```
