基本理解

shell命令和程序

Shell 命令 是由 shell 直接实现的命令，主要用于控制 shell 环境。

程序是独立的可执行文件，功能更复杂，用途更广泛。

使用 type 命令可以判断一个命令是 shell 命令还是程序。

Shell 命令

  • 定义：
    shell 命令是直接由 shell（如 bash、zsh）解释和执行的命令。
  • 特点：
    • 通常是 shell 内置的命令
    • 执行速度快，因为不需要启动新的进程
    • 主要用于控制 shell 的行为或操作 shell 环境。
  • 示例：
命令 作用
cd 改变当前工作目录
echo 输出文本到终端
export 设置环境变量
source 执行脚本文件并在当前 shell 环境中生效
alias 创建命令别名
unalias 删除命令别名
set 设置 shell 选项
unset 取消设置变量或函数
read 从标准输入读取数据
exit 退出当前 shell

程序

  • 定义：
    程序是独立的可执行文件，通常存储在 /bin、/usr/bin 等目录中。
  • 特点：
    • 由 shell 调用并运行在新的进程中
    • 功能更复杂，通常用于完成特定任务。
    • 可以是系统自带的工具，也可以是用户安装的软件。

通配符与正则表达式

一句话总结区别：通配符（wildcards）是文件名匹配模式，正则表达式（regex）是字符串匹配规则

通配符（Wildcards）

  • 特点

    • 用在 Shell（bash）或某些命令（ls、find） 中。

    • 用来匹配文件名。

  • 常见通配符

通配符 含义 示例
* 匹配任意字符（0个或多个） *.txt 匹配 a.txt, b.txt
? 匹配任意单个字符 ?.txt 匹配 a.txt, b.txt
[abc] 匹配括号内任意一个字符 file[12].txt 匹配 file1.txt, file2.txt

常见通配符

正则表达式（Regex）

  • 特点
    • 是 一种更强大的文本匹配规则，常用于grep/sed/awk/perl/Python脚本等
  • 常见正则符号
正则符 含义 示例
. 任意一个字符 a.c 匹配 abc、a1c
.* 任意长度任意字符 a.*c 匹配 abc、axyzc
^ 行首 ^hello 匹配以 hello 开头的行
$ 行尾 world$ 匹配以 world 结尾的行
[a-z] 匹配 a 到 z 之间的任意字符 [0-9]+ 匹配数字

shell命令的单引号和双引号

  • 单引号

当你使用单引号包裹字符串时，Shell 会严格地保留其中的字符原样，这意味着不对其中的任何特殊字符进行扩展或替换。特别是其中的 $、*、? 等特殊字符也不会被展开。

  • 双引号

当你使用双引号包裹字符串时，Shell 会对其中的一些特殊字符（如 $、反引号 等）进行扩展，但是对于 \*、? 和 [] 这些字符，Shell 会 进行匹配，而不会被“保留原样”。

ls/find/grep匹配

ls和find是使用通配符进行匹配，但是ls和find运行时对通配符解释的时机不同；

grep是使用正则表达式进行匹配

假设有三个文件：file1.txt file2.txt file3.log

📌 ls *.txt

这个命令运行时，其实是 shell 做了这件事：

ls 程序根本看不到 *.txt，它接收到的是两个具体文件名，是 shell 提前帮你匹配好了。

📌 find . -name '*.txt'

这个命令运行时，shell只是把 *.txt 这个字符串原样传给 find，find 自己再去查：

find 内部才去判断哪些文件名和 '*.txt' 匹配，它使用自己的匹配机制（不是正则，是 shell-style 匹配）。

NOTE：find . -name *.txt

不加引号，shell 会先把 *.txt 展开成 file1.txt file2.txt

📌 grep 'D*p' file3.log

这个命令运行时，shell只是把 a*c 这个字符串原样传给 grep，让grep自己再去做正则匹配；

通常加引号，防止正则符号被 shell 误解释

Setting APT source file

修改APT源文件需要root权限；

  • sed：
    • sed 是一个流编辑器（stream editor），用于对文本进行处理和转换。
  • -i：
    • 表示直接修改文件内容（in-place edit）。
  • "s/archive.ubuntu.com/mirrors.tuna.tsinghua.edu.cn/g"：
    • s 表示替换操作（substitute）。
    • archive.ubuntu.com 是要查找的字符串。
    • mirrors.tuna.tsinghua.edu.cn 是要替换成的字符串。
    • g 表示全局替换（global），即替换文件中所有匹配的字符串。
  • /etc/apt/sources.list：
    • 这是 Ubuntu 系统中存储软件源地址的文件。

终端复用工具

tmux

tmux- terminal multiplexor

terminal上面只需要运行tmux一个进程即可；

基本介绍

tmux创建出来的进程以及界面布局会保存到session

用户关闭terminal窗口的时候，tmux进程也会退出，但是session中的进程会继续运行；下次进入tmux可以重新选择session，可以恢复原来的窗口布局；

tmux使用

tmux的所有命令都是通过ctrl+b触发；

不带参数的tmux调用:

tmux会自动创建一个新的session，并且创建一个默认的shell。

带有参数的tmux调用：

命令：

左右分屏：ctrl+b+%

上下分屏：ctrl+b+"

切换焦点：ctrl+b+方向键

新窗口：如果当前屏幕的显示已经太多，使用ctrl+b+c开启新的windows;

切换窗口：ctrl+b+窗口号

detached session ：ctrl+b+d，或者直接关闭terminal，此时session中运行的进程并不会终止

session查看：tmux ls列出所有的session

attached session：tmux a -t session号，如果只有一个session，可以省略-t 选项

退出session：

  • 逐个退出进程exit之后，session自动会退出；
  • tmux kill-session -t session号
