数据类型

在C语言中，没有真正的布尔数据类型，而是用一个整数来代替，0代表false，其它值代表true

无符号整数类型size_t

size_t 是 C 语言标准库中定义的无符号整数类型，用于表示对象的大小，定义在 <stddef.h> 或 <stdio.h> 里。

💡 size_t 的具体类型 因平台而异：

平台 size_t 类型 sizeof(int) 的类型
32 位系统 unsigned int unsigned int
64 位系统 unsigned long unsigned long

通过如下方式验证：

  • C语言中sizeof()返回的是size_t；
  • size_t在printf中需要使用%zu格式化描述符；

基础语法

break

在 C 语言中，break 语句用于 立即终止当前循环（for/while/do-while）或 switch 语句的执行

switch

最好有default分支；

每个case最后要有break语句，否则会发生fall-through现象；

case语句可以理解为地址的标识符，所以如果匹配到某一个case之后，如果没有break就会一直向下执行；

default 不是必须的，但建议加上 default 以防止遗漏情况：

  • 避免逻辑错误：如果变量值未匹配任何 case，default 可以提供一个兜底行为。
  • 增强可读性：告诉读者 “如果所有 case 都不匹配，会执行 default”。
  • 便于调试：可以在 default 里打印 unexpected value，帮助发现问题。

字符串&字符串数组

对于C语言来说：

  • \0是一个特殊字符；在存储中的数值为0；字符串常量是以\0结尾；%s输出，以\0为结束；
  • C只是简单地将字符串储存为字符数组，并且在结尾加上\0（空字符）

思考如下问题:

上面两种结果，sizeof输出的结果符合预期，但是%s的输出怎么解释？
🔍 C标准:
C标准对字符串的定义如下（C99 §6.4.5）：

A string literal is a sequence of characters enclosed in double quotation marks, optionally containing escape sequences, and terminated by the null character (\0).

而在使用 %s 的时候，C标准规定（C17 §7.21.6.1）：

If no precision is specified, the array shall contain a null character, and the characters are written up to (but not including) the terminating null character.

==》如果字符数组没有以 \0 结尾，printf 会继续读取内存，直到偶然遇到一个 \0，这可能导致程序崩溃、输出乱码或其他不可预测的行为

原因 1：内存布局的巧合

  • 在你的程序中，full_name 数组之后的内存恰好是 \0。
  • 例如，编译器可能在 full_name 数组之后分配了一块初始化为 0 的内存，或者 full_name 数组之后的内存恰好是 \0。
  • 这种情况下，printf 会“幸运地”遇到 \0，从而正确终止输出。

原因 3：操作系统的内存管理

  • 操作系统在分配内存时，可能会将新分配的内存初始化为 0（尤其是在调试环境中）。
  • 例如，Linux 的 glibc 在某些情况下会将堆内存初始化为 0。

C程序结构

库函数文件头

  • #是预处理指令符号；

  • include是预处理指令，用于包含头文件；

  • <>表示编译器在标准库中查找头文件

main函数的参数

  • argc：

    • 表示命令行参数的数量（argument count）
    • 类型为 int。
    • 至少为 1，因为第一个参数是程序本身的名称。

  • argv：

    • 表示命令行参数的数组（argument vector）

    • 类型为 char *[]，即字符串数组。

    • argv[0] 是程序名称，argv[1] 到 argv[argc-1] 是用户传递的参数

main函数的返回值

返回给操作系统 作为程序的 退出状态码（Exit Status）。

用于判断程序是否成功执行，通常：

  • 返回 0 表示 程序运行成功（Success）。
  • 返回非 0 表示 程序运行失败（Failure），不同的非零值可以代表不同的错误类型。

NULL

NULL的定义：

C 的NULL定义：#define NULL ((void *)0)

  • 0：
    • 在 C 语言中，0 是一个整数类型的数值
  • (void *)：
    • void * 是一种通用指针类型，可以指向任何类型的数据。
  • (void *)0：
    • 将整数 0 转换为 void * 类型的空指针。
    • 空指针表示：不指向任何有效内存地址的指针
    • 这种表示方式明确告诉编译器，NULL 是一个指针，而不是整数

为什么需要 (void *)0？

  • 类型安全：
    • 在 C 语言中，指针和整数是不同的类型。虽然 0 可以表示空指针，但直接使用 0 可能会导致类型不明确。
    • 使用 (void *)0 可以明确表示 NULL 是一个指针。
  • 通用性：
    • void * 是一种通用指针类型，可以隐式转换为任何其他指针类型。
    • 因此，(void *)0 可以安全地用于所有指针类型。

C++的NULL定义：#define NULL 0

在 C 语言中，void * 是一个“通用指针”，几乎可以隐式转换为其他任意指针类型。

但是C++强调类型安全，void * 并不能直接指向其他指针类型

因此C++标准中添加了特性：整数 0被解释为空指针常量

C++11新标准NULL

引入了专用的空指针常量nullptr，解决0是整数还是指针的二义性问题；
