注重基本概念理解

eHSM简介

eHSM是设备的“根信任”：在信息安全中，根信任是整个系统的信任基础。eHSM作为设备的根信任，负责确保关键的安全功能（如加密、密钥管理）与设备的其他部分硬件隔离开来，从而防止未经授权的访问或篡改。

完整的硬件单元：eHSM拥有独立的硬件模块，包括：

  • 时钟（clock）：控制eHSM操作的时序。
  • 复位（reset）：在系统启动或重启时重置硬件状态。
  • 启动（boot）和总线系统（bus system）：负责管理系统的初始化和各硬件模块之间的数据传输。
  • CPU和内存：内置的CPU和内存专门用于处理与安全相关的任务。
  • 加密算法引擎系统：高性能的加密引擎，用于执行复杂的加密算法（如TRNG、HASH、SKE和PKE）。

每次eHSM被重置（例如设备启动时），它都会对自身的硬件系统进行自检，确保所有的硬件组件运行正常。这是为了确保eHSM自身的可靠性和安全性。

安全服务：eHSM为设备（如SoC，系统级芯片）提供多种信息安全相关的服务，包括：

  • 安全存储：保护敏感信息（如密钥、证书等）的存储。
  • 安全启动：确保设备从启动到运行的过程中不受未经授权的干扰。
  • 安全生命周期管理：管理设备的不同安全状态（例如设备的制造、部署、使用和退役阶段）。
  • 密钥管理：安全生成、存储和分发加密密钥。

通信机制：设备（SoC）通过邮箱（mailbox）机制与eHSM通信，使用其提供的加密服务（如TRNG随机数生成、哈希计算、对称和非对称加密）。此外，eHSM还提供了其他硬件接口，允许设备传输传感器数据、状态信号（例如警告和错误信号）等。

Sensor Response

在信息安全领域，传感器通常指用于检测环境或系统中异常情况的设备。具体到eHSM或SoC（系统级芯片），传感器可以监测温度、电压、物理入侵或其他异常状态。当某个传感器检测到异常时，系统可以根据这些位的设置做出不同的响应，比如重置系统、清除密钥或停止加密服务。这些响应的设置确保系统在受到攻击或发生故障时能够采取保护措施。

secure Debug

eHSM secure debug architecture如下图所示：

eHSM通过o_ehsm_status[17:16]控制eHSM和SoC的debug port;

eHSM和SoC的debug port enable/disable依赖于lifecycle

如果eHSM或者SoC debug port处于disable的时候（因为lifecycle的states），需要鉴权才能打开，此鉴权操作可以通过UART或者mailbox进行。

具体鉴权流程如下：

The secure debug authentication supports either asymmetric or symmectric algorithm.

1.asymmetric: SHA256-SECP256R1, SM3-SM2

2.symmetric: AES-CMAC, SM4-CMAC

The secure debug flow is

1.Host sends debug request to eHSMthrough either UART or mailbox

2.eHSMsends challenge (UID + random number) toHost

3.Host sign the challenge with the debug key, and send back toeHSM

​ (a)asymmetric: sign with corresponding private key of ehsm debug key/soc debug key

​ (b)symmetric: calculate the CMAC value with ehsm debug key/soc debug key

4.eHSM verify the signiture with ehsm debug key/soc debug key

5.eHSM assert the o_ehsm_status corresponding bit to enable the eHSM/SoC debug port if verify pass.
